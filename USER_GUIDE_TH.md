# คู่มือใช้งาน TradeLab Paper Bot

TradeLab เป็นบอทสำหรับทดลองกลยุทธ์และเรียนรู้การจัดการความเสี่ยง รองรับ `BTC/USD`, `ETH/USD`, `AAPL` และ `SPY`

> `BTC/USD` และ `ETH/USD` ใช้ราคา Kraken Spot จริงผ่าน Public WebSocket ส่วน `AAPL` และ `SPY` ยังเป็นราคา Synthetic เงินและคำสั่งทั้งหมดเป็น Paper ไม่มีการเชื่อมบัญชีโบรกเกอร์และไม่ส่งคำสั่งด้วยเงินจริง

## 1. เปิดโปรแกรมครั้งแรก

เปิด PowerShell แล้วใช้คำสั่งต่อไปนี้:

```powershell
cd G:\bot_trade
python -m venv .venv
.venv\Scripts\Activate.ps1
pip install -r requirements.txt
uvicorn app.main:app --reload --port 8765
```

เมื่อเห็นข้อความนี้ แสดงว่า server พร้อมแล้ว:

```text
Uvicorn running on http://127.0.0.1:8765
```

เปิดเว็บเบราว์เซอร์ไปที่ <http://127.0.0.1:8765>

ห้ามปิด PowerShell ระหว่างใช้งาน เพราะการปิดหน้าต่างจะทำให้ server หยุดทำงาน

## 2. เปิดโปรแกรมในครั้งถัดไป

ถ้าติดตั้ง dependencies แล้ว ไม่จำเป็นต้องติดตั้งซ้ำ:

```powershell
cd G:\bot_trade
.venv\Scripts\Activate.ps1
uvicorn app.main:app --reload --port 8765
```

เมื่อต้องการหยุด server ให้กลับไปที่ PowerShell แล้วกด `Ctrl+C`

## 3. เริ่มใช้งานบอท

1. เลือกสินทรัพย์จากช่อง **MARKET**
2. ตรวจหรือตั้งค่าในส่วน **Risk & strategy**
3. กด **Save parameters** เมื่อมีการแก้ไขค่า
4. กด **Start bot**
5. ตรวจว่า ENGINE เปลี่ยนเป็น `RUNNING`
6. รอระบบสะสมราคาให้เพียงพอก่อนสร้างสัญญาณ

ค่าเริ่มต้นใช้ Slow SMA 21 และต้องมีข้อมูลทั้งหมด 22 ticks ระบบจึงจะแยกสัญญาณ crossover ได้ แต่ละ tick ห่างประมาณ 2 วินาที จึงใช้เวลารอเริ่มต้นประมาณ 44 วินาที

กด **Stop bot** เพื่อหยุดการสุ่มตัวอย่างราคาเข้ากลยุทธ์และหยุดการตัดสินใจ ราคา Kraken บนหน้าจออาจยังอัปเดตต่อ ระบบจะเก็บยอดเงิน ตำแหน่ง และประวัติไว้ใน `trading.db`

## 4. ความหมายของข้อมูลบนหน้าจอ

### แถบสถานะตลาด

| รายการ | ความหมาย |
|---|---|
| MARKET | สินทรัพย์ที่บอทกำลังติดตามและใช้ตัดสินใจ |
| LAST PRICE | ราคาจริงจาก Kraken สำหรับคริปโต หรือราคาจำลองสำหรับหุ้น |
| BOT SIGNAL | สถานะหรือคำสั่งล่าสุดจากกลยุทธ์ |
| ENGINE | `RUNNING` เมื่อบอททำงาน และ `STOPPED` เมื่อหยุด |

### สรุป Portfolio

| รายการ | ความหมาย |
|---|---|
| Portfolio value | เงินสดรวมมูลค่าตลาดของตำแหน่งทั้งหมด |
| Available cash | เงินจำลองที่ยังใช้เปิดสถานะได้ |
| Open positions | จำนวนสินทรัพย์ที่กำลังถืออยู่ และสัดส่วน exposure |
| Completed trades | จำนวนคำสั่ง BUY/SELL ที่บันทึกไว้ |
| Equity curve | การเปลี่ยนแปลงของมูลค่า Portfolio ตามเวลา |

### ความหมายของ BOT SIGNAL

| Signal | ความหมาย |
|---|---|
| WAITING | ยังไม่เริ่มทำงานหรือเพิ่งรีเซ็ต |
| WARMING UP | กำลังเก็บราคาให้ครบตามช่วง SMA |
| DATA WAIT | Feed กำลังเชื่อมต่อหรือข้อมูลเก่าเกินกำหนด ระบบจะไม่เทรด |
| WATCH | ยังไม่มีสถานะและยังไม่พบจังหวะซื้อ |
| BUY | Fast SMA ตัด Slow SMA ขึ้นและบอทซื้อ |
| HOLD | มีสถานะอยู่และยังไม่มีเงื่อนไขขาย |
| SELL | Fast SMA ตัด Slow SMA ลงและบอทขายทั้งหมดของสินทรัพย์นั้น |
| STOP LOSS | ราคาลดถึงระดับตัดขาดทุน จึงขายทั้งหมด |
| TAKE PROFIT | ราคาขึ้นถึงเป้ากำไร จึงขายทั้งหมด |

## 5. ตั้งค่ากลยุทธ์และความเสี่ยง

| ค่า | หน้าที่ | ค่าเริ่มต้น |
|---|---|---:|
| Fast SMA | จำนวนราคาที่ใช้คำนวณค่าเฉลี่ยระยะสั้น | 8 |
| Slow SMA | จำนวนราคาที่ใช้คำนวณค่าเฉลี่ยระยะยาว | 21 |
| Order size | เปอร์เซ็นต์ของ Portfolio ที่ใช้ต่อการซื้อหนึ่งครั้ง | 10% |
| Max exposure | มูลค่าถือครองสูงสุดต่อสินทรัพย์ เทียบกับ Portfolio | 30% |
| Stop loss | เปอร์เซ็นต์ขาดทุนจากราคาซื้อเฉลี่ยที่สั่งขาย | 4% |
| Take profit | เปอร์เซ็นต์กำไรจากราคาซื้อเฉลี่ยที่สั่งขาย | 8% |

กฎสำคัญ:

- Fast SMA ต้องน้อยกว่า Slow SMA
- Order size ต้องไม่มากกว่า Max exposure
- ค่าที่แก้ไขจะมีผลเมื่อกด **Save parameters** แล้วเท่านั้น
- การแก้ค่าจะไม่ลบตำแหน่งหรือประวัติเดิม
- สำหรับการทดลองครั้งแรก แนะนำให้ใช้ค่าเริ่มต้นก่อน

## 6. วิธีทำงานของกลยุทธ์

บอทใช้ SMA crossover:

1. คำนวณค่าเฉลี่ยระยะสั้นจาก Fast SMA
2. คำนวณค่าเฉลี่ยระยะยาวจาก Slow SMA
3. เมื่อเส้นระยะสั้นตัดขึ้นเหนือเส้นระยะยาว บอทจะซื้อ
4. เมื่อเส้นระยะสั้นตัดลงใต้เส้นระยะยาว บอทจะขายสถานะทั้งหมดของตลาดที่เลือก
5. ระหว่างถือสถานะ บอทตรวจ Stop loss และ Take profit ทุก tick
6. ทุกการซื้อถูกจำกัดด้วย Order size, Available cash และ Max exposure

ระบบไม่ได้รับประกันว่าจะมีคำสั่งซื้อทันที เพราะต้องรอให้เกิด crossover ในชุดราคาที่สะสม

## 7. ตาราง Recent trades

| คอลัมน์ | ความหมาย |
|---|---|
| TIME | เวลาที่คำสั่งเกิดขึ้น |
| ASSET | สินทรัพย์ที่ซื้อหรือขาย |
| SIDE | `BUY` หรือ `SELL` |
| QUANTITY | จำนวนหน่วยที่ซื้อหรือขาย |
| PRICE | ราคาตลาดที่ใช้คำนวณคำสั่ง Paper |
| REASON | เหตุผล เช่น SMA crossover, Stop loss หรือ Take profit |
| P/L | กำไรหรือขาดทุนที่รับรู้เมื่อขาย |

รายการ BUY จะแสดง P/L เป็นขีด เพราะกำไรหรือขาดทุนยังไม่เกิดขึ้นจริงจนกว่าจะขาย

## 8. เปลี่ยนตลาด

เลือกตลาดใหม่จากช่อง MARKET ได้ทั้งตอนหยุดและตอนทำงาน แต่เพื่อให้อ่านผลได้ชัดเจน แนะนำให้กด **Stop bot** ก่อนเปลี่ยนตลาดแล้วค่อยกด **Start bot** อีกครั้ง

ประวัติราคา SMA แยกตามตลาด และตำแหน่งเดิมจะยังคงอยู่จนกว่ากลยุทธ์ของตลาดนั้นจะกลับมาทำงานและพบเงื่อนไขขาย หรือจนกว่าจะรีเซ็ต Portfolio

## 9. รีเซ็ต Portfolio

กดปุ่ม `↻` ด้านข้างปุ่ม Start/Stop แล้วกดยืนยัน การรีเซ็ตจะ:

- หยุดบอท
- คืนเงินจำลองเป็นยอดเริ่มต้น
- ลบตำแหน่งทั้งหมด
- ลบประวัติคำสั่งและ Equity curve
- ล้างข้อมูลราคาที่สะสมไว้

การรีเซ็ตย้อนกลับไม่ได้

## 10. ปรับเงินเริ่มต้นและความเร็ว

คัดลอก `.env.example` เป็น `.env`:

```powershell
Copy-Item .env.example .env
```

แก้ไข `.env` เช่น:

```dotenv
STARTING_CASH=100000
TICK_SECONDS=2
DATABASE_PATH=trading.db
```

| ตัวแปร | หน้าที่ |
|---|---|
| STARTING_CASH | เงินจำลองเริ่มต้น |
| TICK_SECONDS | ระยะห่างระหว่างการสร้างราคาแต่ละ tick |
| DATABASE_PATH | ตำแหน่งไฟล์ฐานข้อมูล |

หลังแก้ `.env` ให้หยุด server ด้วย `Ctrl+C` แล้วเปิดใหม่ หากฐานข้อมูลเดิมมีสถานะค้างอยู่ ให้กด Reset เพื่อเริ่มด้วยยอดเงินใหม่

## 11. แก้ปัญหาที่พบบ่อย

### WinError 10013 หรือเปิดพอร์ต 8000 ไม่ได้

พอร์ต `8000` อาจถูกโปรแกรมอื่นใช้งาน ให้ใช้พอร์ต `8765`:

```powershell
uvicorn app.main:app --reload --port 8765
```

แล้วเปิด <http://127.0.0.1:8765>

### เปิดเว็บแล้วขึ้นว่าเข้าไม่ได้

- ตรวจว่า PowerShell ที่รัน Uvicorn ยังเปิดอยู่
- ตรวจ URL และพอร์ตให้ตรงกับข้อความ `Uvicorn running on ...`
- ลองเปิด <http://localhost:8765>
- ถ้ามี error ใน PowerShell ให้แก้ error นั้นก่อนรีเฟรชหน้าเว็บ

### PowerShell ไม่ยอมรัน Activate.ps1

ใช้คำสั่งนี้เฉพาะหน้าต่าง PowerShell ปัจจุบัน แล้วลอง activate ใหม่:

```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
.venv\Scripts\Activate.ps1
```

### คำสั่ง uvicorn ไม่ถูกพบ

ตรวจว่าหน้าคำสั่งมี `(.venv)` แล้วติดตั้ง dependencies:

```powershell
python -m pip install -r requirements.txt
python -m uvicorn app.main:app --reload --port 8765
```

### บอททำงานแต่ไม่มีรายการซื้อขาย

- รอให้สถานะ WARMING UP สิ้นสุด
- การเกิด crossover ไม่ได้เกิดทุกครั้ง จึงอาจต้องรอต่อ
- ตรวจว่า ENGINE เป็น RUNNING
- อย่าตั้ง Fast SMA มากกว่าหรือเท่ากับ Slow SMA

### ต้องการเริ่มใหม่ทั้งหมด

ใช้ปุ่ม `↻` ใน Dashboard วิธีนี้ปลอดภัยกว่าการลบไฟล์ฐานข้อมูลด้วยตนเอง

## 12. ข้อจำกัดและความปลอดภัย

- ราคา BTC/ETH มาจาก Kraken Spot จริง แต่ราคา AAPL/SPY ยังเป็นข้อมูลจำลอง
- ต้องเชื่อมต่ออินเทอร์เน็ตเพื่อรับราคา Kraken; เมื่อ feed หลุดหรือข้อมูลเก่า ระบบจะหยุดตัดสินใจและแสดง `DATA WAIT`
- ผลกำไรจากระบบจำลองไม่สะท้อนสภาพคล่อง ค่าธรรมเนียม และ slippage จริง
- กลยุทธ์ SMA อาจขาดทุนได้ โดยเฉพาะตลาดที่แกว่งออกข้าง
- อย่านำ API key หรือรหัสผ่านมาใส่ใน source code
- ควรทำ Backtest และทดสอบกับบัญชี Paper ของโบรกเกอร์ก่อนพิจารณาเงินจริง
- ซอฟต์แวร์นี้เป็นเครื่องมือเพื่อการศึกษา ไม่ใช่คำแนะนำทางการเงิน
